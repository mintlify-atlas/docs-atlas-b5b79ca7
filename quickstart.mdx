---
title: "Get started with the Purchase Manager"
description: "Clone the repo, configure your database and JWT settings, spin up the .NET 7 API, and run the Angular 15 frontend in under ten minutes."
---

This quickstart walks you through every step needed to run the Purchase Manager locally: setting up the .NET 7 Minimal API backend with SQL Server, then launching the Angular 15 frontend so you can log in and manage purchases right away.

<Steps>
  <Step title="Prerequisites">
    Make sure the following tools are installed before you begin.

    | Tool | Minimum version |
    |------|----------------|
    | Node.js | 18.x |
    | Angular CLI | 15.x (`npm install -g @angular/cli@15`) |
    | .NET SDK | 7.0 |
    | SQL Server | 2019 or later (or Azure SQL) |

    <Tip>
      Verify your versions by running `node -v`, `ng version`, and `dotnet --version` in your terminal.
    </Tip>
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/AnthonyPortillaCano/FullstacktNet7Angular15.git
    cd FullstacktNet7Angular15
    ```
  </Step>

  <Step title="Configure the backend">
    Open `SolutionCleanArchitecture/CleanArchitecture.MinimalApi/appsettings.json` and update the values for your environment:

    ```json appsettings.json
    {
      "ConnectionStrings": {
        "ConnectionSqlServer": "data source=YOUR_SERVER;initial catalog=YOUR_DB;user id=YOUR_USER;password=YOUR_PASSWORD;TrustServerCertificate=True;"
      },
      "APP_URL_ANGULAR": "http://localhost:4200",
      "LOGGER_URL": "https://apiloggerfinal.azurewebsites.net",
      "Auth": {
        "Jwt": {
          "Issuer": "http://localhost:51239/",
          "Audience": "http://localhost:51239/",
          "Key": "your_strong_secret_key_here",
          "TokenExpirationInMinutes": "20"
        }
      }
    }
    ```

    <Warning>
      Replace `Key` with a long, random string before running in any shared or production environment. The default value in the repository is not secure.
    </Warning>

    The `APP_URL_ANGULAR` value must match the URL where the Angular dev server is served (default: `http://localhost:4200`). The backend uses this value to configure CORS, so any mismatch will block requests from the frontend.
  </Step>

  <Step title="Run the backend API">
    Navigate to the Minimal API project and start the server.

    <CodeGroup>

    ```bash dotnet CLI
    cd SolutionCleanArchitecture/CleanArchitecture.MinimalApi
    dotnet run
    ```

    ```bash dotnet watch (auto-reload)
    cd SolutionCleanArchitecture/CleanArchitecture.MinimalApi
    dotnet watch run
    ```

    </CodeGroup>

    The API listens on `https://localhost:7271` by default.

    <Note>
      Swagger UI is available at `https://localhost:7271/swagger`. Use it to explore all endpoints and send authenticated requests by clicking **Authorize** and supplying a Bearer token obtained from the login endpoint.
    </Note>
  </Step>

  <Step title="Configure the Angular frontend">
    Open `crud-app-web/src/environments/environment.development.ts` and confirm the `apiUrl` points to your running backend:

    ```typescript environment.development.ts
    export const environment = {
      production: false,
      apiUrl: "https://localhost:7271"
    };
    ```

    If you changed the backend port or are running behind a reverse proxy, update `apiUrl` to match. The production environment file (`environment.ts`) points to the deployed Azure endpoint and does not need to be changed for local development.
  </Step>

  <Step title="Run the frontend">
    Install dependencies and start the Angular dev server.

    <CodeGroup>

    ```bash npm
    cd crud-app-web
    npm install
    npm start
    ```

    ```bash Angular CLI
    cd crud-app-web
    npm install
    ng serve
    ```

    </CodeGroup>

    The app is served at `http://localhost:4200`.
  </Step>

  <Step title="Log in and explore">
    Open `http://localhost:4200` in your browser and log in with your SQL Server database credentials.

    Once authenticated, you can:

    - Create, read, update, and delete purchase records
    - Export data to PDF or Excel using the built-in toolbar actions
    - Browse and test the API directly via Swagger UI at `https://localhost:7271/swagger`

    <Tip>
      JWT tokens expire after 20 minutes by default. Increase `TokenExpirationInMinutes` in `appsettings.json` during development to avoid frequent re-logins.
    </Tip>
  </Step>
</Steps>
