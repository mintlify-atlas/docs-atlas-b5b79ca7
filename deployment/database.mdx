---
title: "Create and configure the SQL Server database"
description: "Create the BDEmpresa SQL Server database with the Compra, DetalleCompra, and Users tables needed before the Purchase Manager API can serve requests."
---

The Purchase Manager application stores all of its data in a SQL Server database named **BDEmpresa**. The Entity Framework Core context (`BdempresaContext`) maps to three tables: `Compra` (purchase header), `DetalleCompra` (purchase line items), and `Users` (application accounts). You must create these tables before the API can serve requests. The instructions below apply to both a local SQL Server instance and Azure SQL.

<Warning>
  The `Users` table stores passwords as plain text. This is a significant security risk. Before deploying to production, implement password hashing (for example, using `BCrypt` or ASP.NET Core's `IPasswordHasher<T>`) so that raw passwords are never persisted to the database.
</Warning>

## Create the tables

Run the following DDL statements against your target database. Execute them in order so that the foreign key constraint on `DetalleCompra` can reference `Compra`.

<Steps>
  <Step title="Create the Compra table">
    `Compra` holds one row per purchase. `Id` is an auto-incrementing identity primary key.

    ```sql
    CREATE TABLE Compra (
        Id              INT            IDENTITY(1,1) NOT NULL,
        NumeroDocumento VARCHAR(10)    NULL,
        RazonSocial     VARCHAR(60)    NULL,
        Total           DECIMAL(10, 2) NULL,
        CONSTRAINT PK_Compra PRIMARY KEY (Id)
    );
    ```
  </Step>

  <Step title="Create the DetalleCompra table">
    `DetalleCompra` holds the individual line items for each purchase. The foreign key `FK_IdVenta` links each detail row back to its parent `Compra`.

    ```sql
    CREATE TABLE DetalleCompra (
        Id       INT            IDENTITY(1,1) NOT NULL,
        IdCompra INT            NOT NULL,
        Producto VARCHAR(60)    NULL,
        Precio   DECIMAL(10, 2) NULL,
        Cantidad INT            NULL,
        Total    DECIMAL(10, 2) NULL,
        CONSTRAINT PK_DetalleCompra PRIMARY KEY (Id),
        CONSTRAINT FK_IdVenta FOREIGN KEY (IdCompra)
            REFERENCES Compra (Id)
    );
    ```
  </Step>

  <Step title="Create the Users table">
    `Users` stores application user accounts. The primary key name `PK__Users__3214EC074104FB25` matches the constraint name generated by EF Core scaffolding; you may use any valid name.

    ```sql
    CREATE TABLE Users (
        Id             INT            IDENTITY(1,1) NOT NULL,
        Email          VARCHAR(100)   NULL,
        Password       VARCHAR(128)   NULL,
        FullName       VARCHAR(100)   NULL,
        DocumentNumber VARCHAR(20)    NULL,
        Address        VARCHAR(200)   NULL,
        PhoneNumber    VARCHAR(20)    NULL,
        Token          VARCHAR(200)   NULL,
        CONSTRAINT PK__Users__3214EC074104FB25 PRIMARY KEY (Id)
    );
    ```
  </Step>
</Steps>

## Table reference

The three tables and their columns at a glance:

<AccordionGroup>
  <Accordion title="Compra">
    | Column | Type | Notes |
    |---|---|---|
    | `Id` | `INT IDENTITY` | Primary key, auto-increment |
    | `NumeroDocumento` | `VARCHAR(10)` | Document number |
    | `RazonSocial` | `VARCHAR(60)` | Vendor or company name |
    | `Total` | `DECIMAL(10,2)` | Purchase total |
  </Accordion>

  <Accordion title="DetalleCompra">
    | Column | Type | Notes |
    |---|---|---|
    | `Id` | `INT IDENTITY` | Primary key, auto-increment |
    | `IdCompra` | `INT` | FK → `Compra.Id` (`FK_IdVenta`) |
    | `Producto` | `VARCHAR(60)` | Product name |
    | `Precio` | `DECIMAL(10,2)` | Unit price |
    | `Cantidad` | `INT` | Quantity |
    | `Total` | `DECIMAL(10,2)` | Line total |
  </Accordion>

  <Accordion title="Users">
    | Column | Type | Notes |
    |---|---|---|
    | `Id` | `INT IDENTITY` | Primary key, auto-increment |
    | `Email` | `VARCHAR(100)` | User email address |
    | `Password` | `VARCHAR(128)` | Stored password (hash recommended) |
    | `FullName` | `VARCHAR(100)` | Display name |
    | `DocumentNumber` | `VARCHAR(20)` | Identity document number |
    | `Address` | `VARCHAR(200)` | Street address |
    | `PhoneNumber` | `VARCHAR(20)` | Contact phone |
    | `Token` | `VARCHAR(200)` | JWT token (session cache) |
  </Accordion>
</AccordionGroup>

## Connection string

The API reads the connection string from `ConnectionStrings:ConnectionSqlServer` in `appsettings.json`. The format is:

```json
{
  "ConnectionStrings": {
    "ConnectionSqlServer": "data source=tcp:<server>,1433;initial catalog=BDEmpresa;user id=<user>;password=<password>;TrustServerCertificate=True;"
  }
}
```

In production, set this value as an environment variable or Azure App Service application setting rather than storing credentials in the file.

## Azure SQL vs local SQL Server

<Tabs>
  <Tab title="Azure SQL">
    <Steps>
      <Step title="Create an Azure SQL server and database">
        ```bash
        az sql server create \
          --name purchase-manager-sql \
          --resource-group my-resource-group \
          --location eastus \
          --admin-user sqladmin \
          --admin-password "<strong-password>"

        az sql db create \
          --resource-group my-resource-group \
          --server purchase-manager-sql \
          --name BDEmpresa \
          --service-objective S0
        ```
      </Step>

      <Step title="Allow your client IP through the firewall">
        ```bash
        az sql server firewall-rule create \
          --resource-group my-resource-group \
          --server purchase-manager-sql \
          --name AllowMyIp \
          --start-ip-address <your-ip> \
          --end-ip-address <your-ip>
        ```
      </Step>

      <Step title="Connect and run the DDL">
        Use SQL Server Management Studio (SSMS), Azure Data Studio, or `sqlcmd` to connect to `<server>.database.windows.net` and run the CREATE TABLE statements above against the `BDEmpresa` database.
      </Step>

      <Step title="Update the connection string">
        Replace the `data source` in your connection string with the fully qualified Azure SQL server hostname:

        ```text
        data source=tcp:purchase-manager-sql.database.windows.net,1433;initial catalog=BDEmpresa;user id=sqladmin;password=<password>;TrustServerCertificate=True;
        ```
      </Step>
    </Steps>

    <Tip>
      Enable **Azure Active Directory authentication** on the Azure SQL server to avoid storing SQL passwords in your configuration. You can then use managed identity from your App Service to authenticate without a username and password.
    </Tip>
  </Tab>

  <Tab title="Local SQL Server">
    <Steps>
      <Step title="Create the database">
        Connect to your local SQL Server instance with SSMS or `sqlcmd` and create the `BDEmpresa` database.

        ```sql
        CREATE DATABASE BDEmpresa;
        USE BDEmpresa;
        ```
      </Step>

      <Step title="Run the DDL">
        Paste and execute the three `CREATE TABLE` statements from the section above.
      </Step>

      <Step title="Update the connection string">
        Point the connection string at your local instance. Use Windows authentication or a SQL login.

        ```text
        data source=localhost;initial catalog=BDEmpresa;integrated security=True;TrustServerCertificate=True;
        ```

        Or with a SQL login:

        ```text
        data source=localhost;initial catalog=BDEmpresa;user id=sa;password=<password>;TrustServerCertificate=True;
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>
